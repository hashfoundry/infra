## üéØ **–ß—Ç–æ —Ç–∞–∫–æ–µ service mesh?**

**Service mesh** ‚Äî —ç—Ç–æ –∏–Ω—Ñ—Ä–∞—Å—Ç—Ä—É–∫—Ç—É—Ä–Ω—ã–π —Å–ª–æ–π –¥–ª—è —É–ø—Ä–∞–≤–ª–µ–Ω–∏—è –∫–æ–º–º—É–Ω–∏–∫–∞—Ü–∏–µ–π –º–µ–∂–¥—É –º–∏–∫—Ä–æ—Å–µ—Ä–≤–∏—Å–∞–º–∏, –æ–±–µ—Å–ø–µ—á–∏–≤–∞—é—â–∏–π security —á–µ—Ä–µ–∑ mTLS, observability —á–µ—Ä–µ–∑ distributed tracing, traffic management —á–µ—Ä–µ–∑ intelligent routing, reliability —á–µ—Ä–µ–∑ circuit breakers, policy enforcement —á–µ—Ä–µ–∑ RBAC –∏ automated certificate management –±–µ–∑ –∏–∑–º–µ–Ω–µ–Ω–∏—è –∫–æ–¥–∞ –ø—Ä–∏–ª–æ–∂–µ–Ω–∏–π —á–µ—Ä–µ–∑ sidecar proxy pattern.

## üèóÔ∏è **–û—Å–Ω–æ–≤–Ω—ã–µ –∫–æ–º–ø–æ–Ω–µ–Ω—Ç—ã service mesh:**

### **1. Data Plane (Sidecar Proxies)**
- **Envoy Proxy**: High-performance proxy –¥–ª—è traffic interception
- **Traffic Management**: Load balancing, circuit breaking, retries
- **Security**: mTLS termination, certificate management
- **Observability**: Metrics collection, distributed tracing

### **2. Control Plane**
- **Configuration Management**: Service discovery, routing rules
- **Certificate Authority**: Automated certificate lifecycle
- **Policy Enforcement**: Security policies, access control
- **Telemetry Collection**: Metrics aggregation, monitoring

### **3. Service Mesh Features**
- **Zero-trust Security**: Identity-based authentication
- **Traffic Splitting**: Canary deployments, A/B testing
- **Fault Injection**: Chaos engineering, resilience testing
- **Rate Limiting**: Traffic throttling, DDoS protection

# –§—É–Ω–∫—Ü–∏—è –Ω–∞—Å—Ç—Ä–æ–π–∫–∏ namespace –¥–ª—è service mesh
configure_namespaces_for_mesh() {
    log "üè∑Ô∏è –ù–∞—Å—Ç—Ä–æ–π–∫–∞ namespaces –¥–ª—è service mesh"
    
    # –í–∫–ª—é—á–µ–Ω–∏–µ sidecar injection –¥–ª—è production namespaces
    local namespaces=("production" "staging" "monitoring" "argocd")
    
    for ns in "${namespaces[@]}"; do
        if kubectl get namespace $ns &>/dev/null; then
            log "–í–∫–ª—é—á–µ–Ω–∏–µ sidecar injection –¥–ª—è namespace: $ns"
            kubectl label namespace $ns istio-injection=enabled --overwrite
        else
            log "–°–æ–∑–¥–∞–Ω–∏–µ namespace $ns —Å sidecar injection"
            kubectl create namespace $ns
            kubectl label namespace $ns istio-injection=enabled
        fi
    done
    
    log "‚úÖ Namespaces –Ω–∞—Å—Ç—Ä–æ–µ–Ω—ã –¥–ª—è service mesh"
}
